## 4. 任务一 — 详细计划

### 4.1 前置条件
- [ ] 确认 SSH / JupyterLab 可以访问 GiteeAI 实例
- [ ] `maca-smi` 可以显示 C500 GPU
- [ ] Python 版本为 3.12

### 4.2 安装步骤

**安装 Python 包（脚本 `01_install_deps.sh`）**：
```bash
pip install paddlepaddle==3.4.0.dev20251223 \
    -i https://www.paddlepaddle.org.cn/packages/nightly/cpu/

pip install paddle-metax-gpu==3.3.0.dev20251224 \
    -i https://www.paddlepaddle.org.cn/packages/nightly/maca/

python -m pip install -U "paddleocr[doc-parser]"

pip install opencv-contrib-python-headless==4.10.0.84
```

**克隆并编译 FastDeploy（脚本 `02_build_fastdeploy.sh`）**：
```bash
# 克隆代码
git clone https://gitee.com/paddlepaddle/FastDeploy.git ~/FastDeploy
cd ~/FastDeploy
git checkout release/2.5

# 配置 MACA 环境变量
export MACA_PATH=/opt/maca
[ ! -d ${HOME}/cu-bridge ] && ${MACA_PATH}/tools/cu-bridge/tools/pre_make
export CUCC_PATH=/opt/maca/tools/cu-bridge
export CUCC_CMAKE_ENTRY=2
export CUDA_PATH=${HOME}/cu-bridge/CUDA_DIR
export PATH=${CUDA_PATH}/bin:${MACA_PATH}/mxgpu_llvm/bin:${MACA_PATH}/bin:${CUCC_PATH}/tools:${CUCC_PATH}/bin:${PATH}
export LD_LIBRARY_PATH=${CUDA_PATH}/lib64:${MACA_PATH}/lib:${MACA_PATH}/mxgpu_llvm/lib:$LD_LIBRARY_PATH

# FastDeploy 专用标志
export MACA_VISIBLE_DEVICES="0"
export PADDLE_XCCL_BACKEND=metax_gpu
export FLAGS_weight_only_linear_arch=80
export FD_MOE_BACKEND=cutlass
export ENABLE_V1_KVCACHE_SCHEDULER=1
export FD_ENC_DEC_BLOCK_NUM=2
export FD_SAMPLING_CLASS=rejection

bash build.sh
```

期望输出产物：`~/fastdeploy/dist/fastdeploy-*.whl`

**验证安装（脚本 `03_verify_install.sh`）**：
```bash
pip install ~/fastdeploy/dist/fastdeploy-*.whl
python -c "import fastdeploy; print(fastdeploy.__version__)"
pip show fastdeploy | grep Location
```

## 5. 任务二 — 详细计划

### 5.1 阶段一：性能分析报告

**截止时间**: 在组委会阶段一截止日期前提交  
**提交位置**: PR 至 `PaddlePaddle/community` 的 `rfcs/FastDeploy/` 目录

#### 环境搭建
1. 克隆 FastDeploy `release/2.4`，与已安装的 Paddle + paddle-metax-gpu 配合使用
2. 下载 PaddleOCR-VL-1.5 模型（HuggingFace 或模型中心）
3. 准备典型推理输入集（文档图片，多种尺寸）

#### Profiling 步骤
1. 确认镜像中可用的 MACA profiling 工具（可能是 `mxprof` 或 MACA 适配版 nsys）
2. 运行 `task2-optimization/profiling/run_profile.sh`：
   - 执行 50–100 次推理迭代
   - 收集 GPU 时间线 trace（`.json` 或等效格式）
   - 记录逐算子耗时
3. 将 trace 文件保存到 `task2-optimization/profiling/traces/`

#### 报告必须包含的内容
RFC 报告需要回答以下所有问题：
- **推理框架调度分析**：FastDeploy 如何调度 prefill 和 decode 阶段；CPU↔GPU 同步点在哪里？
- **GPU 利用率**：时间线图；prefill 与 decode 阶段各自的利用率；空闲间隙
- **≥5 个 kernel 分析**：每个 kernel 的名称、执行时间、占总时间百分比、理论 vs 实际 occupancy、瓶颈类型（访存/算力/延迟）
- **内存带宽**：关键 kernel 的峰值 BW 与实际 BW 对比
- **识别的主要瓶颈**：指向阶段二的具体算子或调度模式

#### 报告格式
- 中英双语，分为两个文件：`perf-analysis-report_001.md`（英文）+ `perf-analysis-report_001.zh.md`（中文）
- 包含：profiling 工具命令、环境信息、模型参数、trace 文件位置/链接
- PR 标题：`[Metax GPU] PaddleOCR-VL-1.5 性能分析报告 — oldzhu`

### 5.2 阶段二：优化（组委会选拔后启动）

**前置条件**: 收到组委会邮件确认本提交入选阶段二  
**提交位置**: PR 至 `PaddlePaddle/FastDeploy/develop`

#### 候选优化方向
（将根据阶段一的发现确定具体方向，以下为初步预案）

| 优化方向 | 预期收益 | 复杂度 |
|----------|----------|--------|
| Attention + Norm 算子融合 | 中（5–15%） | 中 |
| Prefill/Decode 重叠调度 | 高（10–25%） | 高 |
| 针对 MACA 的自定义 MoE dispatch kernel | 高（10–20%） | 高 |
| C500 cache 访存布局优化 | 中（5–10%） | 中 |
| 调整 `FD_ENC_DEC_BLOCK_NUM` / batch 参数 | 低–中 | 低 |

#### 阶段二 PR 交付物
- `PaddlePaddle/FastDeploy/develop` 中的源码修改
- `benchmark.sh` 结果表格：优化前后的延迟（ms）与吞吐量（tokens/s）对比
- 确认至少一个关键指标提升 ≥20%
- benchmark 运行截图

## 7. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 编译失败（MACA 头文件缺失） | 检查 `maca-smi`，确认 `/opt/maca` 存在；在 MXMACA 开发者社群求助 |
| MACA profiling 工具未知 | 首次登录时浏览 `/opt/maca/bin/`；在 MXMACA 开发者社群询问 |
| 编译时间过长导致 SSH 超时 | 在 `tmux` 中执行 `bash build.sh`，安全 detach |
| 阶段一未入选阶段二 | 专注报告质量，深度分析 ≥5 个 kernel |
| FastDeploy 2.5 与 2.4 版本不兼容 | 任务一明确使用 2.5（符合打卡规范）；任务二 profiling 明确克隆 2.4 |

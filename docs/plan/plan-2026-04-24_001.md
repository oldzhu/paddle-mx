## 4. Task 1 — Detailed Plan

### 4.1 Prerequisites
- [ ] SSH / JupyterLab access to the GiteeAI instance confirmed
- [ ] `maca-smi` shows the C500 GPU
- [ ] Python 3.12 confirmed

### 4.2 Installation Steps

**Install Python packages (script `01_install_deps.sh`)**:
```bash
pip install paddlepaddle==3.4.0.dev20251223 \
    -i https://www.paddlepaddle.org.cn/packages/nightly/cpu/

pip install paddle-metax-gpu==3.3.0.dev20251224 \
    -i https://www.paddlepaddle.org.cn/packages/nightly/maca/

python -m pip install -U "paddleocr[doc-parser]"

pip install opencv-contrib-python-headless==4.10.0.84
```

**Clone and build FastDeploy (script `02_build_fastdeploy.sh`)**:
```bash
# Clone
git clone https://gitee.com/paddlepaddle/FastDeploy.git ~/FastDeploy
cd ~/FastDeploy
git checkout release/2.5

# MACA environment setup
export MACA_PATH=/opt/maca
[ ! -d ${HOME}/cu-bridge ] && ${MACA_PATH}/tools/cu-bridge/tools/pre_make
export CUCC_PATH=/opt/maca/tools/cu-bridge
export CUCC_CMAKE_ENTRY=2
export CUDA_PATH=${HOME}/cu-bridge/CUDA_DIR
export PATH=${CUDA_PATH}/bin:${MACA_PATH}/mxgpu_llvm/bin:${MACA_PATH}/bin:${CUCC_PATH}/tools:${CUCC_PATH}/bin:${PATH}
export LD_LIBRARY_PATH=${CUDA_PATH}/lib64:${MACA_PATH}/lib:${MACA_PATH}/mxgpu_llvm/lib:$LD_LIBRARY_PATH

# FastDeploy-specific flags
export MACA_VISIBLE_DEVICES="0"
export PADDLE_XCCL_BACKEND=metax_gpu
export FLAGS_weight_only_linear_arch=80
export FD_MOE_BACKEND=cutlass
export ENABLE_V1_KVCACHE_SCHEDULER=1
export FD_ENC_DEC_BLOCK_NUM=2
export FD_SAMPLING_CLASS=rejection

bash build.sh
```

Expected output artifact: `~/fastdeploy/dist/fastdeploy-*.whl`

**Verify install (script `03_verify_install.sh`)**:
```bash
pip install ~/fastdeploy/dist/fastdeploy-*.whl
python -c "import fastdeploy; print(fastdeploy.__version__)"
pip show fastdeploy | grep Location
```

## 5. Task 2 — Detailed Plan

### 5.1 Stage 1: Performance Analysis Report

**Deadline**: Submit before organizer's Stage 1 cut-off  
**Submission**: PR to `PaddlePaddle/community` under `rfcs/FastDeploy/`

#### Setup
1. Clone FastDeploy `release/2.4`, install alongside existing Paddle + paddle-metax-gpu
2. Download PaddleOCR-VL-1.5 model (from HuggingFace or model hub)
3. Prepare representative inference input set (document images, multiple sizes)

#### Profiling
1. Identify the MACA profiling tool available in the image (likely `mxprof`, or a MACA-adapted nsys)
2. Run `task2-optimization/profiling/run_profile.sh` to:
   - Execute 50-100 inference iterations
   - Collect GPU timeline trace (`.json` or equivalent)
   - Record per-operator timing
3. Save trace file to `task2-optimization/profiling/traces/`

#### Report Contents (mandatory)
The RFC must answer all of:
- **Inference framework scheduling**: How does FastDeploy dispatch prefill and decode; where are CPU↔GPU synchronization points?
- **GPU utilization**: Timeline graph; utilization% during prefill vs decode; idle gaps
- **≥5 kernel analyses**: For each kernel — name, execution time, % of total, theoretical vs achieved occupancy, likely bottleneck (memory-bound / compute-bound / latency-bound)
- **Memory bandwidth**: Peak BW vs achieved BW for key kernels
- **Identified top bottleneck**: One specific operator or scheduling pattern to target in Stage 2

#### Report Format
- Bilingual (EN + ZH), filed as two files: `perf-analysis-report_001.md` + `perf-analysis-report_001.zh.md`
- Include: profiling tool command, environment, model info, trace file location / link
- PR title: `[Metax GPU] PaddleOCR-VL-1.5 Performance Analysis Report — oldzhu`

### 5.2 Stage 2: Optimization (starts after organizer selection)

**Prerequisite**: Organizer emails confirmation that this submission was selected for Stage 2  
**Submission**: PR to `PaddlePaddle/FastDeploy/develop`

#### Candidate Optimization Directions
(Will be determined based on Stage 1 findings; preliminary ideas listed below)

| Direction | Expected Impact | Complexity |
|-----------|----------------|------------|
| Fuse attention + norm kernels | Medium (5–15%) | Medium |
| Overlap prefill decode scheduling | High (10–25%) | High |
| Custom MoE dispatch kernel for MACA | High (10–20%) | High |
| Optimize memory layout for C500 cache | Medium (5–10%) | Medium |
| Tune `FD_ENC_DEC_BLOCK_NUM` / batch knobs | Low–Medium | Low |

#### Deliverables for Stage 2 PR
- Source code changes in `PaddlePaddle/FastDeploy/develop`
- `benchmark.sh` results table: latency (ms) and throughput (tokens/s) before vs after
- Confirmed ≥20% improvement on at least one key metric
- Screenshots of benchmark runs

## 7. Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Build fails (missing MACA headers) | Check `maca-smi`, verify `/opt/maca` exists; ask Metax dev group |
| MACA profiling tool unknown | Explore `/opt/maca/bin/` on first login; ask in MXMACA developer group |
